Methodology and Code

The public outputs are produced by two structured language-model workflows. One iterates over event PNGs containing the headline, event time, and Bitcoin price chart. The other iterates over a firm list and retrieves evidence from transcripts, filings, and financial news.

Both workflows are controlled by explicit instruction files and Python loop scripts. The instruction files define the decision rules, required output schema, and validation checks. The Python scripts handle iteration, API calls, output-file management, and consistency checks across runs.

Event Workflow

The event pipeline reads a directory of PNG files, extracts each event number from the filename, sends the image and the event instructions to the model, validates the returned classification, and appends the results to cumulative markdown, CSV, and JSONL outputs.

  • The instruction file defines the event categories, certainty rules, and the required markdown and CSV structure.
  • The Python loop handles image loading, prompt construction, retries, output validation, and file appends.

Firm Workflow

The firm pipeline reads an input CSV of listed firms, applies the firm-classification instructions, and writes firm-level classifications into cumulative markdown, CSV, HTML, and raw JSONL outputs. It also stores a manifest and input snapshot so resumed runs stay tied to the same ordered input universe.

  • The instruction file defines the exposure channels, weighting rules, certainty scale, and materiality coding.
  • The Python loop handles row iteration, manifest creation, validation, retries, and report generation.

Why Instructions Matter

Prompt logic is specified explicitly

The public pages are not free-form model outputs. They are tied to written decision trees, output contracts, and consistency rules that govern the classification process.

Why The Python Loop Matters

Iteration and validation are automated

The scripts enforce repeatable looping over events and firms, preserve cumulative outputs, and check that generated records match the expected schema.
